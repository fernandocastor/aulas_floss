\begin{frame}
	\frametitle{Sem coerção (mas com alguma hierarquia)}
    \begin{block}{Hierarquia baseada em mérito}
    \begin{enumerate}
      \item O {\bf \bgreen{criador}} do projeto é um caso especial\vspace{0.1cm}
      \item \textit{Committers} e não-\textit{committers}\vspace{0.1cm}
      \begin{itemize}
         \item Às vezes há vários níveis de \textit{committers}
      \end{itemize}
    \end{enumerate}
    \end{block}\pause

    \begin{block}{``Norma sem sanção é como fogo que não queima''?} 
    % Esse dito é atribuído a Rudolf von Jhering (http://en.wikipedia.org/wiki/Rudolf_von_Jhering)
    \begin{enumerate}
      \item Projetos \textit{open source} têm regras
         \begin{itemize}
         \item Como fazer que sejam seguidas?\vspace{0.1cm}
         \end{itemize}
         \pause
      \item Ditador \tred{{\em benevolente}}?
      % Em um projeto FLOSS, o ditador funciona muito mais como um mediador de
      % conflitos do que como um ditador de fato. Como discutido a seguir, o 
      % motivo para isso é a possibilidade sempre presente de se criar um fork. 
      % Em outras palavras, o ditador não tem nenhum poder de coerção e não
      % pode ser responsabilizado por manter a ordem e garantir que as regras
      % sejam cumpridas. 
      \begin{itemize}
      \item Tem que ser, senão, enFORKa ele\vspace{0.1cm}
      \end{itemize}
      \pause
      \item Reputação é {\bf tudo}
      % De acordo com Eric Raymond em "Homesteading the Noosphere":
      % ...Software is freely shared. This abundance creates a situation in which the 
      % only available measure of competitive success is reputation among one?s peers.
         \begin{itemize}
         \item E as regras costumam ser pragmáticas
         % Como consequência desse pragmatismo, não seguir as regras tem efeitos 
         % negativos práticos e observáveis nos outros participantes do projeto. 
         % Logo, não seguir as regras pode colocar em cheque a competência de um
         % participante, o que pode arranhar sua reputação. Como reputação é tudo,
         % desvios das regras são evitados a todo custo. 
         \end{itemize}
    \end{enumerate}
    \end{block}
\end{frame}

  \frametitle{A importância da \textit{fork}abilidade}
 {\large
  \begin{itemize}
  	\item \textit{Forks} são a \tred{cola fundamental} de um projeto de FLOSS
      \begin{itemize}
	  \item Mantém a balança de poder\vspace{0.1cm}
	  \item {\bf \tred{Toda}} licença de FLOSS permite a criação de \textit{forks}
	  \end{itemize}\vspace{0.4cm}
\pause
	  \item Democratizam ditaduras
        \begin{itemize}
        \item Poder é {\bf dado} ao ``ditador''
	    \end{itemize}\vspace{0.4cm}
	  \item Reduzem as vantagens de fechar o código
        \begin{itemize}
        \item Quando uma corporação compra uma outra
	    \end{itemize}
	  
	  % Segundo Karl Fogel: 
	  % Forks, or rather the potential for forks, are the reason there are no true 
	  % dictators in free software projects. This may seem like a surprising claim,

	  % copy to rule as they see fit. Would not such a king govern very differently

    \item EGCS -- Enhanced/Experimental GNU Compiler System (\byellow{GCC})\vspace{0.2cm}\pause
    % EGCS = The Enhanced/Experimental GNU Compiler System
    % Motivo: devido à grande popularidade do GCC, a FSF tornou-se muito preocupada com a 
    % estabilidade do mesmo, o que significava que a inclusão de novas funcionalidades tornou-se
    % menos prioritária e mais cuidadosa, para evitar prejudicar essa estabilidade. Alguns
    % desenvolvedores, ansiosos por introduzir melhorias no tronco do código do compilador, 
    % várias delas já implementadas ao longo dos anos, resolveram criar o EGCS em 1997, que 
    % inicialmente servia para experimentar com novas funcionalidades ainda não consideradas 
    % maduras o suficiente para entrar no GCC. Dizer que o EGCS é um fork não é inteiramente 
    % preciso. Na verdade, ele é o resultado da integração de diversos forks do GCC. Em 1999 
    % tornou-se o GCC oficial. O GCC até então era desenvolvido em um estilo "Catedral". A 
    % partir daí, passou a empregar a abordagem "Bazar", já usada no desenvolvimento do EGCS.
    
    \item Apache HTTPd (\byellow{NCSA HTTPd})\vspace{0.2cm}\pause
