\textit{{\bf Presentation and packaging} comprise a wide range of tasks, all revolving around the theme of \textbf{\tred{reducing the barrier to entry}}. Making the project \tred{inviting to the uninitiated} means writing user and developer documentation [...]. Many programmers unfortunately \bred{treat this work as being of secondary importance} to the code itself. [...] it's crucial that they (presentation and packaging) be a priority from the very start of the project.}

\begin{frame}

	\begin{block}{Escolha projetos que lhe \tgreen{interessam}, ou seja}
		  \begin{itemize}
                 \item que sejam úteis para você; ou
                 \item que façam algo que você considere importante; ou

            \item Comunidade relativamente pequena, mas \byellow{{\bf muito}} ativa
            \item \tblue{Interesse direto} para mim
              \begin{itemize}
                 \item Além de ensinar Haskell, pesquiso na área
                 \item Gosto de desenvolver programas que manipulam programas
              \end{itemize}
\pause
            \item Muito \tred{complexo}
              \begin{itemize}
                 \item {\large \textbf{Não seria apropriado para esta disciplina!}}
                 \item Funcionamento \tred{bem diferente} de um compilador Java
                 \item Mas com documentação \byellow{razoável}
                   \begin{itemize}
                     \item dependendo da parte que lhe \bred{interessa}
                   \end{itemize}
                 % Para o compilador propriamente dito, há uma quantidade considerável de documentação. 
                 % Embora seja pesada e exija tempo e dedicação, ela definitivamente ajuda a entender
                 % seu funcionamento. Por outro lado, para o interpretador, há pouca documentação e, 
                 % diferentemente do que pode parecer, é um componente bastante complexo, já que
                 % o que ele interpreta é uma linguagem de baixíssimo nível.
              \end{itemize}
		  \end{itemize}
	\end {block}
\end{frame}

\begin{frame}
\frametitle{As Quatro Apresentações do Projeto}
O que deve ser mostrado em cada apresentação:\vspace{0.3cm}
\begin{enumerate}
\item Build funcionando e discussão sobre teconologias envolvidas
  \begin{itemize}
  \item Deve mostrar que as tecnologias não são um entrave severo.
  \item Traduções nesta apresentação ou na próxima.
  \end{itemize}
  \vspace{0.2cm}
\item Estrutura geral do sistema e como ele pode ser modificado.
  \begin{itemize}
  \item Apresentar um patch implementado!
  \end{itemize}
  \vspace{0.2cm}
\item Primeiras contribuições
  \vspace{0.2cm}
\item Apresentação final
\end{enumerate}
