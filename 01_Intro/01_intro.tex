Making the project \tred{inviting to the uninitiated} means writing user and developer documentation [...]. Many programmers unfortunately \bred{treat this work as being of secondary importance} to the code itself. [...] it's crucial that they (presentation and packaging) be a priority from the very start of the project.}

\begin{frame}
	\frametitle{Sobre a escolha dos sistemas-alvo}
	\begin{block}{Escolha projetos que lhe \tgreen{interessam}, ou seja}
		  \begin{itemize}
                 \item que sejam úteis para você; ou
                 \item que façam algo que você considere importante; ou
                 \item que você use em seu estágio ou trabalho; 
                 \item que tenham a ver com sua iniciação científica
		  \end{itemize}
	\end {block}
                 \vspace{0.5cm}
	\begin{block}{Sistemas que usam ferramentas e tecnologias \tgreen{familiares}}
               \begin{itemize}
                 \item Aprender coisas novas é importante, mas o tempo é limitado
                 \item Tecnologias novas implicam em mais {\em overhead}
               \end{itemize}
    \end{block}
\end{frame}
