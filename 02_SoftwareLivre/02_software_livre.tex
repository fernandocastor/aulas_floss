\begin{frame}
	\frametitle{O que é Software Livre?}
	\begin{block}{Estritamente falando}
	      Um programa é {\bf livre} se seus usuários têm as 4 liberdades essenciais:\pause
		  \begin{enumerate}
  		    \item[0] Liberdade de \tred{executar} o programa, para qualquer fim.\vspace{0.1cm}\pause
		    \item[1] Liberdade para \tred{estudar} o funcionamento do programa e \tred{modificá-lo} para fazer o que você \tred{desejar}.\vspace{0.2cm}\pause
		    \item[2] Liberdade para \tred{redistribuir} cópias. \vspace{0.1cm}
		      \begin{itemize}
		      \item Assim, você ajuda seu vizinho. \vspace{0.1cm}\pause
		      \end{itemize}
		    \item[3] Liberdade para \tred{distribuir} cópias de suas \tred{versões modificadas}.
		      \begin{itemize}
		      \item Assim, a comunidade toda se beneficia com suas modificações.
		      \end{itemize}
		  \end{enumerate}
		   \vspace{0.1cm}\pause
		  \bgreen{Acesso ao código fonte} é precondição para as liberdades 1 e 3.
		  % Originalmente: %
			%The freedom to run the program, for any purpose (freedom 0).
			%The freedom to study how the program works, and change it so it does your computing as you wish (freedom 1). 
			%     Access to the source code is a precondition for this.
			%The freedom to redistribute copies so you can help your neighbor (freedom 2).
			%The freedom to distribute copies of your modified versions to others (freedom 3). By doing this you can 
			%     give the whole community a chance to benefit from your changes. Access to the source code is a precondition for this.
			%
		  %"Free as in freedom, not as in free beer"

	\end{block}
	
\end{frame}

\section{Copyleft}	
\subsection{Copyleft}	
%Copyleft (Boa fonte de informação: http://en.wikipedia.org/wiki/Copyleft)
%As quatro liberdades do software liver não garantem que ``a derivative work will be distributed under the same liberal terms". Uma licença de copyleft garante que trabalhos derivados de um que utiliza tal licença sejam distribuídos usando a mesma licença ou outra que seja equivalente. Isso garante que a distribuição de trabalhos derivados respeita as mesmas liberdades que as respeitadas na distribuição do trabalho original. Essas licenças são chamadas de "virais" porque se auto-perpetuam.
\begin{frame}
	\frametitle{Propagação de liberdades}
	\begin{block}{Em resumo}
		  \begin{itemize}
		    \item Usar regras de \textit{copyright} para \bgreen{evitar a restrição de liberdades}
		    \item Um programa ser livre não implica em obras derivadas também serem
		  \end{itemize}
	\end{block}
	\pause
	\vspace{0.2cm}
\begin{columns}
\begin{column}[l]{7cm}
	\begin{block}{\textit{Copyleft}}
		  \begin{itemize}
		    \item Trabalho derivados são {\bf obrigados} a ser distribuídos
		    \begin{itemize}
		    \item Com a mesma licença que o trabalho original; ou
		    \item Com uma outra que seja equivalente. 
		    \end{itemize}
		  \end{itemize}
	\end{block}
\end{column}

\begin{frame}
	\frametitle{Mais sobre \textit{copyleft}}
	%Tirado da Wikipedia (página mencionada acima:)
%The most well known free software license that uses strong copyleft is the GNU General Public License. Free software licenses that use "weak" copyleft include the GNU Lesser General Public License and the Mozilla Public License. Examples of non-copyleft free software licenses include the X11 license, Apache license and the BSD licenses.
% A Mozilla Public License impõe algumas restrições a mais que visam preservar a integridade da marca Mozilla.
	\begin{block}{GNU General Public License (GNU GPL)}
		  \begin{itemize}
		    \item Licença de software livre mais popular
		    \item É uma licença de \textit{copyleft}
		    \begin{itemize}
		    \item Detratores dizem que é uma licença \tred{``viral''}. 
		    \end{itemize}
		  \end{itemize}
	\end{block}
	\vspace{0.2cm}
	\pause
	\begin{block}{Licenças de \textit{copyleft} forte e fraco}
		  \begin{itemize}
		    \item A GPL usa \textit{copyleft} \textbf{forte}\vspace{0.1cm}
		    \item \textit{Copyleft} fraco \bred{não exige que todos} os trabalhos derivados usem a mesma licença
		    \begin{itemize}
		    \item Normalmente, os que usam o trabalho original sem modificá-lo\vspace{0.1cm}
		    %Útil, por exemplo, para bibliotecas. A glibc é distribuída com a LGPL.
		    \item Exs.: GNU Lesser GPL (LGPL) e Mozilla Public License
		    \end{itemize}
		   \end{itemize}
% Uma licença de copyleft fraco aplica as garantias de copyleft a apenas um subconjunto dos trabalhos derivados a partir de um que utiliza uma licença desse tipo. Normalmente é utilizada com bibliotecas, por exemplo, para evitar que um sistema que utiliza uma biblioteca distribuída com uma licença de copyleft fraco sem modificá-la precise também ser distribuído com uma licença de copyleft. O copyleft só se aplica se essa biblioteca tiver sido modificada. 

% Este material de aula, excetuando-se as figuras, utiliza uma licença similar a copyleft. Há algumas diferenças, porém. Em resumo, a licença empregada (CC BY-NC-SA) é mais restritiva, já que proíbe o uso comercial e exige que crédito seja atribuído ao seu autor (sem implicar em endosso de qualquer forma, da parte do autor, a eventuais modificações) quando trabalhos derivados a partir deste forem criados. Neste sentido, ela remove algumas liberdades de pessoas que utilizarem este material. Uma discussão interessante sobre porque NC-SA não é copyleft pode ser encontrada neste endereço: http://robmyers.org/weblog/2008/02/noncommercial-sharealike-is-not-copyleft.html
	\end{block}
